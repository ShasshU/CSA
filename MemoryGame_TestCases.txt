1. TEST PURPOSE
================================================================================

This document validates the MemoryGame.java program, which shows a random
sequence of single-character "memory strings" in GUI buttons one at a time. The
player recreates the sequence from memory. The program uses MemoryGameGUI.java
(the GUI/board class) and RandomPermutation.java (shuffle helper).

Console output observed:

TEST CASE 7: Input Longer Than Sequence
────────────────────────────────────────────────────────────────────────────────
Configuration: Easy level (sequence length 3); player types 5 characters

Steps:
  1. Watch 3-character sequence  (e.g., B A C)
  2. When dialog appears, type "B A C D E"
  3. Click OK

Expected behavior:
  - cleanGuess becomes "BACED" (5 chars) after cleanup
  - cleanGuess.length() (5) != randomSequence.length (3) -> match = false

Console output observed:

  Player guess    : BACED
